## 📄 Sobre el Archivo HTML

El sitio web de RabGamesStudio está construido como un **único archivo HTML autocontenido** (`index.html`). No requiere servidor, base de datos ni dependencias instaladas — solo un navegador moderno.

### Tecnologías usadas

| Tecnología | Uso |
|---|---|
| HTML5 | Estructura semántica de la SPA |
| CSS3 (puro) | Estilos, animaciones, CRT effect, grid, responsive |
| JavaScript (vanilla) | Interactividad completa — slider, modales, FAQ, i18n, formulario |
| Google Fonts CDN | `Press Start 2P` · `Share Tech Mono` · `Rajdhani` |
| Imágenes base64 | Screenshots de His Destiny y Talk to the Hand embebidos en el JS |
| Imágenes CDN (GameJolt) | Thumbnails, avatares y banners cargados por URL |
| Cloudflare Worker | Backend del formulario (email + Discord DM) |

### Requisitos

- **Navegador moderno** (Chrome, Firefox, Edge, Safari 2020+)
- **Conexión a internet** para: Google Fonts, imágenes CDN de GameJolt/Wix, Cloudflare Worker (formulario)
- **Sin conexión:** funciona parcialmente — texto, CRT, slider con placeholder, interactividad JS

### Peso del archivo

- La v4.16 pesa aproximadamente **~540 KB** (incluye imágenes base64 embebidas, sistema bilingüe completo, CRT)

## ✨ Features Completas

### 🌐 Sistema Bilingüe (ES/EN)

- **178+ elementos** traducidos — absolutamente todo el texto del sitio cambia al alternar idioma
- Pop-up de idioma al cargar (primera visita o sin selección previa)
- Toggle de idioma en el header (🇬🇧 ENG / 🇪🇸 ESP) disponible en todo momento
- Implementado con objeto de traducciones `T = { es: {...}, en: {...} }` y función `applyLang(v)`
- Placeholders, labels, botones, toasts, FAQ, políticas legales — todo bilingüe

### 🕹️ CRT Effect

- Botón **CRT: OFF / ON** en el header activa/desactiva el modo retro
- Implementado con CSS puro: `body::before` (scanlines) + `body::after` (viñeta) + `animation: crt-flicker`
- Sin canvas ni WebGL — ligero y compatible con todos los navegadores
- Al activar: el cursor personalizado hace fade-out (0.4s) y aparece el cursor del sistema
- Al desactivar: el cursor del sistema desaparece y el punto rosa hace fade-in

### 🖱️ Cursor Personalizado

- Punto rosa de 8px con `z-index: 999999` — siempre visible por encima del CRT overlay y todos los popups
- Hover: escala 1.6x en color morado
- Click: comprime ligeramente
- Oculto en dispositivos táctiles (`@media (hover: none)`)
- Fade-out suave al activar CRT · fade-in al desactivar

### 🎞️ Slider del Hero

- 3 slides con auto-avance cada 4.8 segundos
- Flechas manuales izquierda/derecha y dots de navegación
- Contenido bilingüe en cada slide

### 🎮 Modal de Juegos

### ⚖️ Políticas Legales (Modal)

- Se abren sin salir de la página — click en footer

## 📬 Formulario de Contacto → Cloudflare Worker

### Flujo de envío

```
Usuario llena formulario
       ↓
FormData enviado a Cloudflare Worker (POST)
  con campo source="rab" (fd.append explícito en JS)
       ↓
Worker identifica fuente → config RabGamesStudio
       ↓
Worker procesa y envía en paralelo:
  ├── 📧 Email HTML estilizado vía Resend → 4 destinatarios
  │     (adjuntos incluidos como attachments en Resend)
  └── 💬 Discord DM con Embed Discohook → IDs configurados
        (adjuntos como embeds separados con imagen inline + spoiler)
```

### Campos del formulario

| Campo HTML | `name` | Tipo | Notas |
|---|---|---|---|
| `#f_name` | `name` | text | Requerido |
| `#f_email` | `email` | email | Requerido |
| `#f_social` | `social` | text | Opcional — nick/usuario |
| `#f_platform` | `socialPlatform` | select | Requerido si `social` tiene valor |
| `#f_type` | `reason` | select | Tipo de consulta |
| `#f_msg` | `message` | textarea | Requerido |
| `#f_attach` | `attachment` | file | Opcional · image/* · múltiples |
| *(hidden)* | `source` | hidden | `value="rab"` — identifica el origen al Worker |

### Plataformas disponibles en el selector de red social

`discord` · `twitter` · `instagram` · `gamejolt` · `itchio` · `bluesky` · `facebook` · `youtube` · `tiktok` · `otro / other`

### Validación

- Si `social` está vacío → `#f_platform_row` permanece oculto
- Si `social` tiene texto → `#f_platform_row` aparece dinámicamente (`oninput`)
- Al intentar enviar: si hay nick sin plataforma → muestra selector, muestra error localizado, detiene el envío
- Al éxito: limpia todos los campos, oculta el selector de plataforma

### Respuesta del Worker

```json
{
  "success": true,
  "results": { "email": true, "discord": true, "source": "RabGamesStudio" },
  "errors": []
}
```

### Sistema de Traducción

```javascript
// Objeto T con todas las cadenas
const T = {
  es: {
    nav_home: 'Inicio', form_name: '// NOMBRE',
    form_platform: '// ¿EN QUÉ RED?', // ...178+ claves
  },
  en: {
    nav_home: 'Home', form_name: '// NAME',
    form_platform: '// WHICH PLATFORM?', // ...178+ claves
  }
};

// Aplicar traducciones
function applyLang(v) {
  const t = T[v];
  document.querySelectorAll('[data-i18n]').forEach(el => {
    if(t[el.dataset.i18n] !== undefined) el.innerHTML = t[el.dataset.i18n];
  });
  document.querySelectorAll('[data-i18n-ph]').forEach(el => {
    if(t[el.dataset.i18nPh] !== undefined) el.placeholder = t[el.dataset.i18nPh];
  });
}
```

## 🚀 Cómo publicar en GitHub Pages / Netlify

### GitHub Pages (método actual)

1. Crea un repositorio en GitHub
2. Asegúrate de que el archivo se llame `index.html`
3. Ve a **Settings → Pages → Source → Deploy from branch → main / root**
4. Tu web estará en `https://TU_USUARIO.github.io/NOMBRE_DEL_REPO/`

Para dominio personalizado: agrega un archivo `CNAME` con tu dominio y configura los DNS.

### Netlify (alternativa recomendada)

1. Ve a https://netlify.com → cuenta gratuita
2. Arrastra `index.html` al dashboard
3. Netlify lo publica instantáneamente
4. En **Site Settings → Domain Management** conecta tu dominio

### Estructura de archivos recomendada

```
/
├── index.html          ← el sitio completo
├── favicon.png         ← ícono del tab y embed de redes
├── README.md           ← este archivo
└── assets/             ← (opcional) imágenes externas al HTML
    └── images/
        ├── ttth_banner.png
        └── hd_banner.png
```

### Conectar el Cloudflare Worker

1. Crea una cuenta en [workers.cloudflare.com](https://workers.cloudflare.com)
2. Crea un nuevo Worker y pega el código de `worker-unified.js`
3. Ve a **Settings → Variables and Secrets** y configura los secrets (ver tabla arriba)
4. En `index.html`, el `WORKER_URL` ya está configurado:
   ```js
   const WORKER_URL = 'https://xata-portfolio-bot.addictivegamer.workers.dev/';
   ```
